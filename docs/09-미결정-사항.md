# 09. 미결정 사항

구현 착수 전 또는 PoC 후 결정이 필요한 항목.

## 정책 결정 (우선)

- [ ] 아젠다 생성 AI 레벨 (룰 / 하이브리드 / LLM 중심) — **전체 구현 일정에 영향**
- [ ] 진척도 평가 방식 디테일 (안건 완료율 자동 계산 + 사용자 보정 범위)
- [ ] 사유 결석 인정 권한 (자동 / 팀장 승인) — **데이터 모델에 영향**

## STT 관련 (RealtimeSTT)

- [ ] **faster-whisper 모델 크기 선택** (`tiny` / `base` / `small`) — 1주차 PoC에서 한국어 정확도·지연 비교 후 결정
- [ ] VAD 임계값 (RealtimeSTT 내장 VAD 파라미터) — PoC 후 결정
- [ ] `confidence` 산출 방식 — faster-whisper 세그먼트 평균 logprob 등 PoC에서 검증
- [ ] **STT 실패 판정 방식** — 활성 윈도우에서 차감하는 `anomaly_events.event_type='stt_failure'`의 트리거 조건 (큐 적체·재시도 실패·연속 결손 구간 등)
- [ ] faster-whisper 모델 가중치 다운로드 호스팅 위치·무결성 검증 방법

## 데이터 모델 후속 반영 (06 공식 반영분)

06 새 공식이 도입한 신규 필드를 03 데이터 모델에 반영해야 함:

- [ ] `meetings.meeting_type` (`regular` / 기타) — 정규 회의만 기여도 산정 대상
- [ ] `action_items.difficulty` (정수 1~3: 쉬움/보통/어려움) — 액션 축 가중치
- [ ] `presence_events.disconnect_classification` (자발 / 비자발) — 활성 윈도우·출석률 분모 보정
- [ ] `anomaly_events.event_type = 'stt_failure'` — 활성 윈도우 차감 트리거
- [ ] `attachments` 테이블 — 액션 아이템에 첨부되는 자료

## 자료 첨부

- [ ] **첨부 파일 저장소** — S3(Start AWS 권한 확인 필요) / EC2 EBS 로컬 / RDS BLOB 중 선택
- [ ] 업로드 방식 — multipart 직접 vs presigned URL
- [ ] 허용 파일 형식·용량 상한

## 1주차 PoC 후 결정

- [ ] 신뢰도 등급 정확한 임계값
- [ ] 마감 준수 점수 곡선 수치

## 디테일 미설계

- [ ] 회의 외 시간 대시보드 레이아웃
- [ ] 회의 후 정리 화면 디테일
- [ ] 회의 전 아젠다 검토 화면 디테일
- [ ] DB 스키마 상세 정의
- [ ] 미접속자 경고 정책 디테일

## 수치 확정

- [ ] 글자수 상한 (현재 500자)
- [ ] 누적 점수 가중 방식 옵션 확정 (시간 가중 추천)
- [ ] 3축 기본 가중치(액션 50 / 발언 30 / 참석 20)의 팀별 조정 허용 범위

## 기타

- [ ] 회의 무효 처리 기준
- [ ] 테스트 모드 회의 (누적 미반영 옵션)
- [ ] 마감일 자동 기본값 (다음 회의 시작 vs 회의 종료 후 N일)
- [ ] 데이터 삭제 요청 권한 (본인 / 팀장)
- [ ] 누적 점수 결석 제외 정책 디테일
- [ ] 외부 게스트 입장 P0 여부
- [ ] 최소 회의 길이 (5분 미만 누적 미반영)
