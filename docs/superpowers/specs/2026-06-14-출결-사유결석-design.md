# 출결 표시 + 사유 결석 동의 — 설계

작성일: 2026-06-14

## 목적

종료된 회의의 **출결(출석/지각/결석)을 시각적으로 확인**할 수 있게 하고, 결석에 대해 **사유를 입력해 팀원 과반이 동의하면 출석으로 인정**(기여도 점수상 불이익 제거)한다.

## 배경 (기존 구조)

- 회의 종료 시 참여자별 `ContributionScore`가 저장된다: `attendance_ratio`(입장 기록 없으면 null), `punctuality_score`(지각 시 감점), `meeting_score` 등.
- 기여도 산정 엔진은 이미 `excused_absence`(사유 결석) 개념을 가진다 — 사유 결석이면 **누적 점수에서 그 회의를 제외**한다(불이익 없음). 단 현재는 추적 수단이 없어 항상 `false`로 전달한다(`contribution.mapper.ts` "사유 결석 미추적").
- 따라서 출결 표시는 추가 저장 없이 저장값에서 파생하고, "출석 인정"은 기존 `excused_absence` 경로에 연결한다.

## 설계 결정 (확정)

1. **출석 인정 시 점수 반영** = 누적 평균에서 그 회의 제외(불이익 제거). 기존 `excused_absence` 메커니즘 재사용 — 산정 계약 변경 없음.
2. **동의 정족수** = 그 결석 건의 본인만 제외한 나머지 현재 활성 팀원의 절반 이상. `필요 동의수 = ceil((현재_활성_팀원수 − 1) / 2)`. 결석자가 여러 명이어도 각 결석 건은 독립적으로, "그 건의 본인만" 모수에서 제외한다(다른 결석자는 동의 가능).
3. **출결 판정** = 저장된 `ContributionScore`에서 파생(추가 계산/저장 없음).
4. **사유 인정 대상** = 결석만. 지각/출석은 표시 전용.
5. **UI** = 회의 상세에 '출결' 탭 신설.

## 데이터 모델 (신규 2개)

```
MeetingAbsence  — 결석 사유 1건 (회의 × 사용자)
  id            PK
  meeting_id    FK
  user_id       FK (결석자 본인)
  reason        text (사유)
  status        enum: pending | approved | rejected
  created_at
  UNIQUE(meeting_id, user_id)   -- 회의당 본인 사유 1건

AbsenceConsent  — 동의 1건 (중복 동의 방지)
  id            PK
  absence_id    FK
  voter_id      FK (동의한 팀원)
  created_at
  UNIQUE(absence_id, voter_id)
```

출결 상태(출석/지각/결석)는 별도 테이블에 저장하지 않고 `ContributionScore`에서 파생한다.

## 출결 판정 (파생 규칙)

| 조건 | 표시 |
|------|------|
| `attendance_ratio == null` (입장 기록 없음) | 결석 |
| 입장 O, `punctuality_score < 1` (팀 설정 유예 초과) | 지각 |
| 그 외 | 출석 |
| 위가 결석이고 해당 회의에 `status=approved`인 `MeetingAbsence` 존재 | 출석 인정(결석) |

- 지각 시간(분)은 `punctuality_score`/원시 presence에서 함께 도출해 표기("지각 12분").
- 유예 기준은 팀 설정 `punctuality_grace_ratio`를 그대로 따른다(산정 로직과 일치).

## 플로우

1. **사유 입력** — 결석으로 판정된 **본인**이 사유 입력 → `MeetingAbsence(status=pending)` 생성.
2. **동의** — 결석자를 제외한 **활성 팀원**이 동의 → `AbsenceConsent` 1건. 멱등(이미 동의했으면 무시).
3. **자동 승인** — 동의 수가 `ceil((활성_팀원수 − 1)/2)`에 도달하면 `status=approved`로 전환하고, **그 회의 기여도를 재산정**한다(`excused_absence=true` 전달 → 누적에서 제외).
4. **시한** — 입력·동의는 회의 **확정(confirm) 전까지만** 가능. 확정 시점에 `pending`인 건은 결석으로 확정(이후 변경 불가).

### 상태 전이

```
(없음) --본인 사유 입력--> pending --동의 정족수 도달--> approved --재산정--> 누적 제외
                            │
                            └--회의 confirm 시 미달--> (결석 확정, 변경 불가)
```

## API (신규)

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/meetings/:id/attendance` | 출결 목록(파생) + 각자 사유/동의 현황 |
| POST | `/api/meetings/:id/absences` | 본인 결석 사유 입력 (종료·미확정 회의, 본인이 결석일 때만) |
| POST | `/api/absences/:id/consent` | 동의 (결석자 제외 팀원, 멱등, 정족수 도달 시 자동 승인+재산정) |

`GET /attendance` 응답 항목(멤버별): `user_id`, `name`, `status(present|late|absent|excused)`, `late_minutes?`, `absence?{ id, reason, consent_count, consent_required, my_consent, approved }`

### 권한·검증
- 사유 입력: 요청자 == 결석자 본인, 회의 status == ended && 미확정, 본인이 실제 결석 판정일 때만.
- 동의: 요청자가 그 팀의 활성 멤버 && 결석자 본인이 아님. 본인 결석 건에 자기 동의 불가.
- 재산정은 기존 `ContributionsService.computeAndStoreMeetingScores`(① 회의 점수) + 팀 집계 경로를 재사용.

## 산정 연결 (excused_absence 채우기)

- `contributions.service.ts`의 ① 산정 입력 구성 시, 그 회의에 `approved` 사유가 있는 사용자는 `excused_absence=true`로 표시해 엔진/로컬 스코어러에 전달한다.
- `contribution.mapper.ts`의 `deriveMemberData`에서 현재 하드코딩된 `excused_absence: false`를 "approved 사유 여부"로 대체한다.
- 팀 집계(②③④)에서도 누적 입력 시 동일하게 반영(엔진은 사유 결석을 누적에서 제외).

## UI (출결 탭)

회의 상세 탭에 `attendance` 추가: `아젠다 · 발언기록 · 출결 · 결정사항 · 회의요약`.

```
출결 현황
 ● 김철수     🟢 출석
 ● 이영희     🟡 지각 12분
 ● 박민수     🔴 결석        [사유 입력]   ← 본인일 때
 ● 정지원     🟢 출석

 — 다른 멤버 결석 건 —
 최서준  🔴 결석 · 사유: 가족행사
            동의 2/3   [동의하기]
```

- 배지 색상은 기존 토큰 재사용: 출석 `--green`, 지각 앰버, 결석 `--coral`. 출석 인정은 초록 배지 + "(인정)" 표기.
- 본인이 결석이면 사유 입력 버튼(모달), 타인 결석 건엔 동의 버튼 + `N/M` 현황. 이미 동의했으면 비활성.
- 회의 확정 후에는 액션 버튼 숨김(읽기 전용).

## 범위 밖 (YAGNI)

- 명시적 반대표 없음 — 동의만 카운트, 정족수 미달이면 `pending` 유지.
- 지각 사유 인정 없음 — 결석만.
- 알림(동의 요청/승인) 연동은 후속 과제.
- `rejected`는 스키마에만 두고 이번 구현에선 자동 전이 없음(향후 명시적 거부용 예약).

## 테스트 관점

- 출결 파생: attendance_ratio null→결석, punctuality<1→지각, 정시→출석, approved 사유→출석 인정.
- 정족수: 5인 팀(결석 1) → 2명 동의 시 승인, 1명이면 미승인.
- 멱등: 동일인 중복 동의 시 카운트 1.
- 승인 시 재산정으로 누적에서 제외되는지(점수 불이익 사라짐).
- 권한: 타인 사유 입력 거부, 본인 자기 동의 거부, 확정 회의 변경 거부.
